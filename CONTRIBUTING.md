# Contributing to ZAPETTE Whitepaper

ZAPETTEおよび関連プロジェクトに関心を寄せていただき、ありがとうございます。
私たちは、**「建設的な評議（Constructive Council）」**というDEAIASの哲学を、コミュニティ運営にも適用したいと考えています。

本リポジトリは、「構想（Concept）」と「仕様（Spec）」を管理するドキュメント専用のリポジトリです。
技術的な指摘だけでなく、ビジネスモデルや倫理的な観点からのフィードバックも歓迎します。

## 🤝 参加方法

### 1. Discussions & Issues
現在、議論の主な場は **[Issues](https://github.com/wee-hours-dev/zapette-whitepaper/issues)** です。

*   **🐛 Bug Report**: ドキュメントの誤字脱字、数式の誤り、リンク切れなど。
*   **💡 Feature Request**: 「こんな機能がSidekickに欲しい」「TRPGモードでこういう遊び方はどうか」といったアイデア。
*   **⚠️ Risk Assessment**: 「この仕様には法的リスクがあるのでは？」「スパム対策が甘いのでは？」といったリスク指摘（Verifierの役割）。

### 2. Pull Requests
ドキュメントへの直接の修正提案も歓迎します。
ただし、大きな変更（ビジネスモデルの根幹に関わる部分など）については、まずはIssueで議論を起案してください。

## 📜 ガイドライン

### "Moderator" Mindset
コメントをする際は、以下の「モデレーター・マインドセット」を意識してください。

1.  **Be Constructive**: 批判する際は、必ず「代替案」や「改善の方向性」をセットで提示してください。
2.  **Respect Context**: 過去の議論や、ドキュメントの前提（Why）を尊重してください。
3.  **Stay Cool**: 感情的な対立ではなく、論理的な整合性を追求しましょう。
