# ZAPETTE ホワイトペーパー: アルゴリズムと公正性

このドキュメントでは、ZAPETTEの核となる「公正な収益分配を実現するアルゴリズム」について解説します。

なお、基盤技術であるマルチAIエージェントシステムについては、別紙 [DEAIAS アーキテクチャ](deaias-architecture.md) を参照してください。

## 1. クオリティ・スコアリングと分配アルゴリズム

ZAPETTEの核となるのは、PV（ページビュー）偏重の評価経済からの脱却です。私たちはコンテンツの質を定量的・定性的に評価し、公正な収益分配を行うために以下の数理モデルを採用します。

### 1.1 評価モデル

コンテンツのスコア ($Score$) は、単一の指標ではなく、複数の変数を組み合わせたモデルによって算出されます。状況やフェーズに応じて、以下の2つのモデルを使い分け、または併用します。

#### A. 加重和モデル (Linear Weighted Model)
各指標のバランスを調整しやすい、安定運用フェーズ向けのモデルです。

$$
Score = w_1 \cdot \hat{P} + w_2 \cdot C + w_3 \cdot T
$$

ここで、
- $\hat{P}$: 正規化されたPV (Normalized PV / Bounded Log PV)
- $C$: 読了率 (Completion Rate)
- $T$: 信頼性スコア (Trust Score)
- $w_1, w_2, w_3$: 各指標の重み係数 ($w_1 + w_2 + w_3 = 1$)

#### B. 加重積モデル (Geometric Weighted Model)
いずれか一つの指標でも著しく低い場合、スコア全体が低くなる、品質重視フェーズ向けのモデルです。

$$
Score = \hat{P}^{a} \times C^{b} \times T^{c}
$$

### 1.2 変数の定義

| 変数 | 定義 | 測定方法 |
| :--- | :--- | :--- |
| **正規化PV ($\hat{P}$)** | バズ依存を抑制したアクセス指標 | 対数スケール、またはシグモイド関数による上限クリッピングを行い、極端なスパイクの影響を緩和します。 |
| **読了率 ($C$)** | コンテンツ消費の深度 | スクロール深度、滞在時間、インタラクション頻度から算出。短時間での離脱はスコアを減衰させます。 |
| **信頼性スコア ($T$)** | 内容の正確性と安全性 | 別途定義されるAI監査（DEAIAS）によるファクトチェック結果と、査読者（ピア）による評価の合算値です。 |

### 1.3 異常値検出と防御 (Anti-Gaming Logic)

クリックファームやBotによる組織的な不正操作（Gaming）を防ぐため、統計的手法によるフィルタリングを実装します。

#### Modified Z-Score (修正Zスコア)
平均値ではなく中央値を用いることで、外れ値に頑健な異常検知を行います。

$$
M_i = \frac{0.6745(x_i - \tilde{x})}{MAD}
$$

- $x_i$: 対象の計測値
- $\tilde{x}$: 中央値 (Median)
- $MAD$: 中央絶対偏差 (Median Absolute Deviation)

判定基準として $|M_i| > 3.5$ となるアクセスや評価は「異常値」と見なし、分配計算から以下のいずれかの処置を行います。
1. **除外**: スコア計算から完全に排除する。
2. **減衰**: 寄与度を著しく低く見積もる（例: $0.1$倍）。
3. **冷却 (Cooldown)**: 一定時間、評価の反映を保留し、詳細な監査キューへ送る。
